PASS 101 — GITHUB-SAFE FILE SIZES
Date: 20 June 2026

The Pass 100 SQLite full-text index stored a second copy of normalized record
content and had grown to approximately 53.2 MB. Pass 101 replaces it with a
contentless FTS5 index plus a small row-ID map. The database JSON files remain
the source of truth and all 1,712 bilingual records are preserved.

The application now detects the compact-v2 index format and joins matched FTS
row IDs to record IDs. A maintenance rebuild tool creates the compact index in a
temporary file, validates the strict 40,000,000-byte ceiling, and atomically
replaces the active index only after a successful build.

A second maintenance tool scans the whole project and exits with an error if any
file reaches the 40,000,000-byte repository limit.
